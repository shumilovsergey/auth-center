# auth-center

Централизованный сервис аутентификации. Приложение перенаправляет пользователя на auth-center, тот проверяет личность и возвращает одноразовый код, приложение меняет код на данные пользователя через server-to-server вызов.

```
Пользователь → твоё приложение → auth-center → (Telegram / Solana / Google / ...)
                                       ↓
                              одноразовый code
                                       ↓
                       твой бэкенд → /exchange → данные пользователя
```

Auth-center не хранит сессии и не имеет базы данных — только верифицирует личность и отдаёт её приложению.

---

## Подготовка сервера
Скопировать сервисные файлы из `bin/example.*.service` в `/etc/systemd/system/`, заполнить все переменные окружения, затем:

```bash
systemctl daemon-reload
systemctl enable --now auth-center
systemctl enable --now auth-client
```

## Dev flow

```
python/   ← MVP, proof of concept, запускается через Docker
go/       ← продакшн, компилируется в статический бинарник под linux/amd64
```

Изменения в Python не переносятся в Go автоматически. Когда фича проверена — рефакторинг в Go делается отдельно.

Бинарники коммитятся в git. На сервере нет сборки — только `git pull` и рестарт сервиса.
